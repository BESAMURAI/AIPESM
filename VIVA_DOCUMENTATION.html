AIPESM

AI-Powered Academic Performance Evaluation & Support System

Project Type: Web Application | Technology: Full-Stack with AI Integration

1. Introduction

AIPESM (AI-Powered Academic Performance Evaluation & Support System) is a comprehensive web-based application designed to revolutionize academic performance tracking and student support. The system provides intelligent insights into student performance while offering personalized learning recommendations through AI-powered analysis.

The platform addresses critical challenges in modern education by automating performance tracking, identifying areas where students need improvement, and providing targeted learning resources based on individual academic weaknesses.

Problem Statement

Traditional academic tracking systems lack intelligent analysis and personalized recommendations. Teachers struggle to identify struggling students early, and students lack access to targeted learning resources based on their specific weaknesses.

Solution

AIPESM integrates AI-powered analysis to automatically identify weak subjects, analyze syllabus content, and recommend personalized learning resources using semantic similarity algorithms and external search APIs.

2. Objectives

  • Develop a user-friendly interface for teachers to manage classes, students, and academic records
  • Implement real-time performance tracking and visualization through interactive charts
  • Create an AI-powered recommendation system that identifies weak subjects and suggests relevant learning resources
  • Design a responsive web application accessible across all devices
  • Ensure secure authentication and role-based access control for teachers and students
  • Integrate machine learning algorithms for intelligent content matching and ranking

5. System Architecture

5.1 Database Schema

The system uses MongoDB with the following collections:

Collections: - users: {email, password_hash, role} - classes: {_id, name, teacher_email} - students: {email, name, class_id} - subjects: {subject, class_id} - marks: {student_email, class_id, subject, marks} - syllabus: {class_id, subject, unit, content} - resources: {subject, type, title, description, link}

5.2 Application Flow

User Authentication Flow:

  1. User submits credentials on login page
  2. System verifies credentials against MongoDB users collection
  3. Session created with user role (teacher/student)
  4. Redirect to appropriate dashboard based on role

AI Recommendation Flow:

  1. System identifies subjects where student marks < 40%
  2. Retrieves syllabus content for identified weak subjects
  3. Builds search query combining subject name and syllabus keywords
  4. Queries SerpAPI to fetch relevant learning resources
  5. Uses TF-IDF vectorization on syllabus and search results
  6. Calculates cosine similarity between syllabus and each resource
  7. Ranks resources by similarity score
  8. Filters out irrelevant results (blacklist filtering)
  9. Returns top 5 most relevant resources to student

6. Key Algorithms & Implementation

6.1 TF-IDF Vectorization

Term Frequency-Inverse Document Frequency (TF-IDF) is used to convert text documents into numerical vectors, enabling mathematical comparison between syllabus content and learning resources.

from sklearn.feature_extraction.text import TfidfVectorizer vectorizer = TfidfVectorizer() vectors = vectorizer.fit_transform([syllabus_text] + resource_texts)

6.2 Cosine Similarity

Cosine similarity measures the cosine of the angle between two vectors, providing a similarity score between 0 and 1. Higher scores indicate more similar content.

from sklearn.metrics.pairwise import cosine_similarity similarities = cosine_similarity(vectors[0:1], vectors[1:]).flatten()

6.3 Syllabus Parsing

Regular expressions are used to parse syllabus text into units, extracting unit names and content for structured analysis.
